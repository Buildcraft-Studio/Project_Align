- The executable target is named Project_Align_Rebuild and is emitted to the Release output folder by CMake.
- Include paths are resolved through the include directory and the runtime prefix style used throughout the codebase.
- The current demo run prints the expected watchdog lifecycle messages, including the worker runtime being marked STALED and then auto-restarted after the DEAD threshold.

Runtime system notes:
- RuntimeManger is assumed to be runtime ID 0.
- Bootstrap registers DebugAndLogRuntime (ID 1) and SupervisorRuntime (ID 2) at startup.
- SupervisorRuntime can create workers at runtime and assign IDs starting at 100.
- WatchDog thresholds:
  - OK when sinceAlive is within the configured tick window
  - STALED when a runtime misses the window
  - DEAD after repeated STALED failures, which triggers a restart path
- Current runtime modules include:
  - Bootstrap
  - DebugAndLogRuntime
  - SupervisorRuntime
  - LuaRuntime (stubbed)
  - RenderRuntime (stubbed)
  - ErrorHandler (stubbed warning sink)

Geometry notes:
- The geometry layer currently contains basic mesh/vector primitives under the Engine::Geometry namespace.
- Mesh2 supports vertex addition and stubbed square/circle generation.
- Modifier is a base class stub intended for future deformation operations.

Known open items:
- DEGRADED status is declared but not yet used by the watchdog logic.
- ErrorHandler::raiseWarning is still an empty stub.
- Lua and rendering integration are not implemented yet.
- The current code is a prototype and should be treated as a foundation for further runtime/engine work.
