Build notes:
- Compiler search directory: `include` only (all includes use the `runtime/` prefix,
  e.g. #include "runtime/runtimeManger.hpp", even from files inside src/runtime/ itself).
- RuntimeManger is assumed to be runtime ID 0. Bootstrap registers DebugAndLogRuntime (ID 1)
  and SupervisorRuntime (ID 2) at startup; SupervisorRuntime hands out IDs 100+ to any
  workers it spawns later, at runtime.
- WatchDog thresholds: OK if sinceAlive <= 100 ticks, STALED per failed check past that,
  DEAD (auto-restart) after 5 consecutive STALED results.
- Known open items: DEGRADED (status 2) is declared but unused; ErrorHandler::raiseWarning()
  is an intentionally empty stub; dynamically-spawned workers only accumulate sinceAlive if
  something calls incrementCounter(id) for them specifically — a central per-tick sweep over
  the whole table hasn't been added yet.
