Detailed Description

Factory/supervisor-style runtime module. Unlike Bootstrap, which registers everything once up front, SupervisorRuntime creates new WorkerRuntime instances DURING normal operation (e.g. one per incoming job) and registers each one with RuntimeManger::setRuntime() (createRuntime) as it's spawned.

  • Note
    SupervisorRuntime owns the workers it creates (unique_ptr) so the raw pointers RuntimeManger holds stay valid for their lifetime, same pattern Bootstrap uses for its modules.
