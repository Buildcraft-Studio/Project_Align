Detailed Description

All runtime will be started and manged by the RuntimeManger Module.

After the Bootstrap module load all the Runtime are loaded, mangagement is handed to RuntimeManger. Modules may also be registered later, at runtime, by a supervisor-style module (see SupervisorRuntime) rather than only at Bootstrap time.

Note
RuntimeManger itself is assumed to be runtime ID 0.

◆ statusCode

enum class Engine::Runtime::RuntimeManger::statusCode
strong

Every status a runtime entry in the table can carry.

OK/STALED/DEAD are computed automatically by WatchDog(). KILLED is a transient step DEAD passes through on its way back to STARTED. STARTED and INACTIVE (14/15) are manual flags — nothing derives them automatically; see setRuntime() and setStatus().

Enumerator
OK 

Runtime passed its WatchDog check within the OK tick window (0-100 ticks).

DEAD 

Runtime failed the WatchDog check 5 times in a row; it is killed and restarted.

DEGRADED 

Reserved: runtime performance starts to lag more than 100 ticks. Not yet driven by WatchDog().

STALED 

Runtime failed a single WatchDog check (elapsed ticks > 100). A warning fires (via ErrorHandler) and a restart is attempted if this repeats.

KILLED 

Set when the RuntimeManger issues a kill order for the runtime (set right before a DEAD restart, or on manual kill).

STARTED 

Manual flag. Set once by Bootstrap (or SupervisorRuntime) when a runtime module is first loaded, before RuntimeManger takes over management of it.

INACTIVE 

Manual flag. When set, the runtime will be skipped when RuntimeManger initiates startup.

◆ setRuntime()

void Engine::Runtime::RuntimeManger::setRuntime ( int16_t id,
std::string setName,
IRuntimeModule * module )

Register a runtime module with the manager so it has a table entry to log status against. This is "createRuntime" - the function that adds an entry to the runtime table.

This is what Bootstrap calls to hand a freshly-loaded module off to RuntimeManger at startup. It can also be called later, at runtime, by a supervisor/factory-style module (see SupervisorRuntime::createWorker()) to register a module it spawns on demand. The entry is created with status STARTED (14) — a manual flag meaning "loaded, but not yet WatchDog-checked" - and this is also the point an ErrorHandler is stood up for this runtime ID.

Parameters
idThe runtime ID Bootstrap/this module identifies itself with.
setNameHuman-readable name for the table/logs.
modulePointer to the module RuntimeManger will poll during WatchDog(). Ownership is NOT taken - the caller must keep the pointed-to module alive for as long as it stays registered.

◆ setStatus()

void Engine::Runtime::RuntimeManger::setStatus ( int16_t id,
statusCode status )

Manually set a runtime's status, e.g. INACTIVE (15) to keep it from starting, or STARTED (14) to re-arm it.

This is the entry point a DebugAndLog-style runtime (or any external caller) uses to WRITE into the table, mirroring getStatus()/getTable() for reads.

Parameters
idThe runtime ID to update. No-op if id isn't registered.
statusThe status to set.
Note
This does not run any restart/kill logic - it only overwrites the logged flag. Use restartRuntime() if you need the counters cleared too.

◆ WatchDog()

void Engine::Runtime::RuntimeManger::WatchDog ( )

Sweep the table: every registered module reports its own ID back, and its record is scored and logged:

  • - sinceAlive in [0, WATCHDOG_TICK_LIMIT] -> OK, failCount reset to 0
    • sinceAlive > WATCHDOG_TICK_LIMIT -> STALED, failCount++, ErrorHandler::raiseWarning() fires
    • failCount reaches WATCHDOG_MAX_FAILURES -> DEAD, restartRuntime() is called automatically
